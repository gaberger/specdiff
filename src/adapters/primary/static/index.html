How to Use apidiff

      
Compare OpenAPI specs across versions to detect breaking changes, renamed fields, type changes, and new additions.


      
1. Load Specs

      
Choose one of three ways to load API specs into the editors:

      
    
        
  • Sidebar Integrations — Click a provider (Stripe, GitHub, Twilio, Forward Networks) to expand. Click a curated comparison or pick two versions from the dropdowns and click "Fetch & Compare".
    • 
        
  • Fetch from URL — Paste an OpenAPI spec URL into the input below each editor and click "Fetch".
    • 
        
  • Paste or Drop — Paste JSON directly into an editor, or drag and drop a .json/.yaml file onto it.
    • 
      


      
2. Review the Diff

      
    
        
  • Filter chips — Click the colored chips (all, breaking, renamed, added, etc.) to filter the results table.
    • 
        
  • Click any path segment in the table to scroll both editors to that location. Each segment is individually clickable.
    • 
        
  • Unified diff column shows additions (+) in green and removals (-) in red.
    • 
        
  • Editor highlighting — Changed lines are colored by type (red=removed, green=added, blue=renamed, amber=changed).
    • 
      


      
3. Editor Controls

      
    
        
  • Scroll lock — The padlock icon between editors toggles synchronized scrolling on/off.
    • 
        
  • Line numbers — Each editor shows line numbers. Version and file size appear below.
    • 
        
  • Syntax highlighting — JSON is highlighted using Shiki (rose-pine-dawn theme) for files under 500 lines.
    • 
      


      
4. Version Picker (Live Fetch)

      
Providers with a version picker (Stripe, GitHub, Twilio, Forward Networks) fetch real published OpenAPI specs from their GitHub repos or documentation sites. Specs are cached on disk for instant reload.


      
5. CLI Usage

      
# Compare local files
bun run start:cli diff old.json new.json

# Compare provider specs by version
bun run start:cli diff --provider forward 26.2 26.3
bun run start:cli diff --provider stripe v2228 v2229

# Generate a migration guide
bun run start:cli guide old.json new.json --base v1 --revision v2


      
6. Algorithm Validation

      
Curated integrations include documented API change notes. The collapsed "Validation" bar shows how many documented changes the diff algorithm detected. Click to expand details.
